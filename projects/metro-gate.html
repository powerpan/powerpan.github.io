Metro Gate Detection

基于 YOLOv5 的地铁闸机翻越检测系统，从模型训练到 RK3588 边缘设备部署的全链路实践，实现地铁场景下违规行为的实时识别与告警。

Overview

地铁安全的 AI 守门人

地铁闸机翻越（或钻越）是城市轨道交通中最常见的违规行为之一，不仅造成票务损失，更存在严重的人身安全隐患。 传统方案依赖人工监控，效率低、漏检率高。本项目基于 YOLOv5 目标检测模型， 训练了一个专门识别闸机翻越行为的检测器，并将其部署到 Rockchip RK3588 边缘设备上， 实现了从视频流输入到违规告警输出的全链路实时推理。

Challenge

场景挑战

地铁闸机场景的视觉检测面临多重挑战：

  • 遮挡严重 — 闸机通道狭窄，人体经常被闸机挡板、其他乘客遮挡，需要模型具备较强的遮挡推理能力
  • 动作多样 — 翻越动作包括跨栏式翻越、撑手翻越、钻越等多种形态，且持续时间短（通常 1-3 秒），对检测的实时性要求高
  • 光照复杂 — 地下站台光照条件多变，存在逆光、侧光、荧光灯频闪等问题
  • 边缘部署 — 需要在功耗受限的嵌入式设备上运行，模型必须足够轻量
Model Design

模型设计

基于 YOLOv5s 架构进行定制化改造。首先对 Backbone 进行轻量化，将 C3 模块中的标准卷积替换为 深度可分离卷积（Depthwise Separable Conv）， 在保持特征提取能力的同时将参数量从 7.2M 降低到 4.1M。

数据集方面，我们从地铁站监控视频中采集了 8,000+ 帧图像，标注了三类目标：normal_pass（正常通行）、 climb_over（翻越闸机）和 crawl_under（钻越闸机）。采用 Mosaic + MixUp + 随机裁剪增强策略， 并针对翻越动作的时序特性，引入了相邻帧的光流特征作为辅助输入。

Hardware

RK3588 硬件平台

Rockchip RK3588 是目前国产边缘 AI 芯片中性能最强的 SoC 之一，集成了四核 Cortex-A76 + 四核 Cortex-A55 CPU、 Mali-G610 GPU 以及独立的 6 TOPS NPU（Neural Processing Unit）。其 NPU 支持 INT8/INT16/FP16 混合精度推理， 特别适合部署量化后的 YOLO 系列模型。

我们选用的开发板配备了 8GB LPDDR5 内存和独立的 NPU 加速单元。相比 GPU 方案（如 Jetson Orin Nano）， RK3588 的优势在于更低的功耗（典型 5-8W vs 15W）和更高的性价比，非常适合地铁站这种需要大规模部署的场景。 NPU 的 6 TOPS 算力对于 YOLOv5s 级别的模型绰绰有余，实测推理速度可达 60+ FPS。

Deployment

部署流程

从 PyTorch 模型到 RK3588 NPU 部署，需要经过以下关键步骤：

  • Step 1: 导出 ONNX — 将 PyTorch 模型导出为 ONNX 格式，固定输入尺寸为 640×640
  • Step 2: RKNN 转换 — 使用 Rockchip 提供的 rknn-toolkit2 将 ONNX 模型转换为 RKNN 格式，进行 INT8 量化（校准集 500 张）
  • Step 3: NPU 部署 — 在 RK3588 上使用 rknn-lite2 运行时加载 RKNN 模型，通过 NPU 加速推理
  • Step 4: 视频流对接 — 使用 OpenCV 采集 RTSP 视频流，逐帧送入模型推理，结果通过 WebSocket 推送至告警系统
Optimization

推理优化

在 RK3588 上部署时，我们进行了多维度的推理优化：

  • INT8 量化 — 使用 rknn-toolkit2 的量化功能，将 FP32 模型量化为 INT8，模型体积缩小 4 倍，推理速度提升 2.5 倍，mAP 仅下降 1.2%
  • 多核 NPU 并行 — RK3588 的 NPU 支持 3 核并行推理，通过设置 core_mask=RKNN.NPU_CORE_0_1_2 充分利用算力
  • 零拷贝优化 — 使用 DMA buffer 实现视频帧到 NPU 的零拷贝传输，减少内存拷贝开销
  • 异步流水线 — 视频采集、预处理、推理、后处理四个阶段采用异步流水线，最大化吞吐量

RKNN 推理代码

rknn_infer.py
from rknnlite.api import RKNNLite
import cv2, numpy as np
rknn = RKNNLite()
rknn.load_rknn('yolov5s_gate.rknn')
rknn.init_runtime(core_mask=RKNNLite.NPU_CORE_0_1_2)
# Preprocess
frame = cv2.imread('gate_frame.jpg')
blob = cv2.resize(frame, (640, 640))
blob = blob[:, :, ::-1].transpose(2, 0, 1).copy()
# NPU inference
outputs = rknn.inference(inputs=[blob])
# Post-process: NMS + decode
