概述

为什么需要 StackForge？

每个新项目都要重新搭建脚手架——选择框架、配置 ESLint、设置 CI/CD、编写 Dockerfile、配置环境变量……这些重复劳动消耗了大量宝贵时间。

StackForge CLI 将这些最佳实践封装成可组合的模板，一条命令即可生成生产级项目骨架。核心引擎用 Rust 编写，确保极速执行；模板系统基于 Node.js 生态，保持灵活性。

支持 50+ 种技术栈组合，从 React + Go 到 Next.js + Rust，从移动端到 CLI 工具，覆盖主流开发场景。内置 CI/CD 模板支持 GitHub Actions、GitLab CI、Jenkins 等主流平台。

使用示例

快速上手

只需一条命令，即可初始化一个完整的全栈项目。StackForge 会交互式引导你选择技术栈，并自动生成所有配置文件。

terminal
$ npm install -g @stackforge/cli
$ stackforge init my-app --template nextjs-rust
⚡ StackForge v2.4.0
✓ Scaffolding project structure...
✓ Installing dependencies (Next.js 14, Rust 1.74)
✓ Generating CI/CD pipeline (GitHub Actions)
✓ Creating Docker configuration (multi-stage)
✓ Setting up Terraform modules
🎉 Project ready!
cd my-app && stackforge dev
stackforge.config.ts
import { defineConfig } from '@stackforge/core';
export default defineConfig({
name: 'my-app',
template: 'nextjs-rust',
features: [
'docker',
'github-actions',
'terraform',
],
database: 'postgresql',
auth: 'clerk',
});
