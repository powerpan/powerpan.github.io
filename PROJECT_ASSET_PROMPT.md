# Project Asset Prompt

这是一份给 AI 助手使用的项目整理提示。无论你是哪个 AI 助手，你的任务都不是把整个代码仓库机械复制出来，而是把当前项目整理成一份可长期复用的项目资产文档。

默认产出位置：

```txt
当前项目根目录/
```

默认产出文件名：

```txt
PROJECT_ASSET.md
```

如果项目根目录已经有类似文件，例如 `PROJECT_ASSET.md`、`PROJECT_CONTEXT.md`、`PROJECT_REVIEW.md` 或 `README.md` 中已有项目资产内容，优先合并更新，不要制造重复文档。

## 工作原则

先理解项目，再沉淀资产。

项目资产文档应该帮助未来的我快速理解这个项目：它解决什么问题、现在做到哪里、技术栈是什么、核心模块有哪些、做过哪些关键决策、踩过哪些坑、沉淀出了哪些可复用资产。

不要默认要求能访问整台电脑，也不要假设存在个人知识库目录。只整理当前项目中能够看到的内容。

不需要运行测试，不需要启动项目，不需要重新构建索引。重点是把项目内容整理好、组织好、写清楚。

不要把这些目录作为主要阅读对象：

- `.venv/`
- `node_modules/`
- `dist/`
- `build/`
- `.git/`
- `__pycache__/`
- 缓存文件、临时文件、大型二进制文件

优先阅读这些材料：

- `README.md`
- `requirements.txt` / `pyproject.toml` / `package.json`
- `.env.example`
- `docs/`
- `scripts/`
- 核心入口文件
- 主要配置文件
- 现有项目说明、复盘、架构文档

## 整理方法

请按下面顺序工作：

1. 扫描当前项目根目录，了解项目类型和入口。
2. 阅读 README、依赖文件、配置文件、脚本和核心源码。
3. 判断项目的真实目标、当前状态和主要工作流。
4. 在当前项目根目录创建或更新 `PROJECT_ASSET.md`。
5. 保留已有重要内容，不要覆盖用户手写判断。
6. 把新增内容写成清晰段落，方便未来复制到知识库或 RAG 系统。
7. 对不确定的信息标注“待确认”。

## 写作要求

- 文件名清晰可读。
- 内容短而密，不写流水账。
- 用当前项目里的具体事实，不写空泛模板话。
- 优先沉淀判断、决策、踩坑和可复用资产。
- 不要粘贴完整源码。
- 如果需要引用代码，只引用关键路径和简短说明。
- 不确定的信息标注“待确认”。
- 不要求运行测试或验证命令。
- 不要求访问当前项目之外的目录。

## 完成后的自检

整理完成后检查：

- 是否在当前项目根目录产出了 `PROJECT_ASSET.md` 或等价文档
- 是否说明项目背景和当前状态
- 是否包含技术栈、核心模块、关键流程
- 是否记录关键决策、问题、踩坑经验
- 是否整理出可复用资产
- 是否提出下一步建议
- 是否避免复制完整源码和依赖目录内容
- 是否把不确定信息标注为“待确认”
